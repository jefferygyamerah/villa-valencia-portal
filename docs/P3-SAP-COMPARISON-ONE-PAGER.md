# P3-SAP-001 — SAP Comparison One-Pager

Date: 2026-05-01  
Product: Villa Valencia / APROVIVA  
Purpose: buyer-facing positioning against SAP-style enterprise maintenance systems.

## Positioning statement

APROVIVA is not trying to be SAP. SAP is an enterprise asset-management backbone. APROVIVA is a focused condo/HOA operating layer that turns daily building work into board-ready evidence.

The useful comparison is not feature parity. It is fit:

- **SAP:** deep, standardized enterprise maintenance and finance integration for large organizations.
- **APROVIVA:** lightweight, Spanish, phone-first, board-governance-oriented operations for PH/condo associations.

## Side-by-side

| Capability | SAP-style maintenance / ERP | APROVIVA |
| --- | --- | --- |
| Target customer | Enterprise asset-heavy organizations | Condos, PHs, HOA boards, property managers |
| Core workflow | Notification → order → planning → execution → confirmation → analytics | Hallazgo/incidente → assignment → action → evidence → closure → Junta report |
| User experience | Powerful but complex; planner/operator roles require training | Phone-first for conserjería; simple role views for supervisor, gerencia, junta |
| Board governance | Usually custom reporting/BI on top | Native Junta view: decisions, escalation, chronic patterns, backlog, capital projects |
| Mobile field capture | Available through enterprise apps, often heavier | Built around tap/choose/photo/point workflows |
| Accounting integration | Strong enterprise finance integration | Not full accounting; complements accounting with operational evidence |
| Maintenance planning | Deep asset structures, work centers, task lists, preventive plans | Plan Maestro, recorridos, inspection points, findings, work assignments |
| Inventory | Mature materials/MM integration | Lightweight stock counts, reorder alerts, procurement status |
| Reporting | Embedded analytics/BI, highly configurable | Board-ready drill-downs and planned PDF packet for HOA governance |
| Implementation | High configuration effort, process redesign, training | Static app + Supabase; quick vertical deployment for a building/community |
| Best use | Large enterprise standardization | Condo operations accountability and board transparency |

## APROVIVA’s advantage

1. **Condo-specific governance**  
   The product speaks in Junta, Gerencia, conserjería, recorridos, hallazgos, proveedores, and board decisions — not generic enterprise work centers.

2. **Board packet from operations**  
   APROVIVA’s commercial edge is not another dashboard. It is the ability to generate board-ready reporting from daily operational evidence.

3. **Phone-first field adoption**  
   Conserjería can work from a simple mobile flow: choose the plan, tap the point, register the exception, add photo if needed.

4. **Lightweight deployment**  
   A community can start with one building, seed data, and controlled workflows without a heavy ERP project.

5. **Transparent accountability**  
   Every issue should have origin, owner, due date, status, evidence, and history — the discipline buyers associate with enterprise systems, but packaged for PH operations.

## SAP’s advantage

Be honest in sales conversations:

- Mature enterprise asset hierarchy.
- Deep finance/procurement/materials integration.
- Enterprise-grade security/roles/workflows.
- Broad ecosystem and standard reporting.
- Better fit for large multi-site industrial/enterprise maintenance.

## Do not claim

Avoid these claims:

- “APROVIVA replaces SAP.”
- “APROVIVA has full enterprise asset management.”
- “APROVIVA is a complete accounting system.”
- “APROVIVA is security-equivalent to enterprise ERP today.”

Use instead:

- “APROVIVA borrows the lifecycle discipline of enterprise maintenance and packages it for condos.”
- “APROVIVA complements accounting by explaining what happened operationally.”
- “APROVIVA is a focused vertical operating layer for boards and building teams.”

## Buyer answer: why not just SAP or generic ERP?

For a PH/condo, SAP is usually too heavy for the problem. The daily problem is not lack of enterprise ERP; it is that building work lives in chats, spreadsheets, photos, and meeting minutes.

APROVIVA solves the near-term pain:

- field capture,
- owner/status/due date,
- evidence trail,
- board drill-down,
- operating report.

If a buyer already has SAP or another ERP, APROVIVA can still sit as the field/board layer that structures condo-specific operations and later exports/integrates summarized records.

## Product roadmap implications

To strengthen enterprise credibility without bloating the product:

1. Keep lifecycle vocabulary consistent: intake, assignment, execution, confirmation, closure, analytics.
2. Add asset/component fields gradually for reserve/capital linkage.
3. Add board packet PDF before deep ERP-style configuration.
4. Harden auth/RLS before external pilots.
5. Keep finance positioning narrow: operational evidence and board reporting, not full accounting replacement.
